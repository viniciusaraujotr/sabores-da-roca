🍯 Geleia de Mocotó — Modo de preparo
📋 Ingredientes (1 panela)
Mocotó13 kg
Açúcar10 a 11 kg
Açúcar p/ queimar0,5 kg
Ácido cítrico5 colheres
Sorbato5 colheres
Canela em pó1 colher
Cravo/canela/erva-doceA gosto
Água (liquidificador)~1 baldinho/batida
0 de 10 etapas concluídas
Colocar 13kg de mocotó na panela de pressão com cravo, canela e/ou erva-doce. Cozinhar por 1h15min após ferver.
⏱ 1h15min
Queimar 0,5kg de açúcar separadamente para dar cor. Reservar.
Bater no liquidificador — ~4 batidas, 1 baldinho de água por batida. Deve sobrar mocotó na última peneirada.
⏱ ~5min
Posicionar peneira diretamente sobre o tacho e peneirar o mocotó batido direto.
Adicionar 10 a 11kg de açúcar ao tacho com o mocotó já peneirado.
Acender o fogo — não pode faltar até o final. Incorporar açúcar queimado enquanto mexe.
Verificar ponto: tigela com água, mexer 7x no tacho, colher, escorrer, rolar na tigela, passar dedo.
⏱ ~5min
Ao encontrar a liga ideal, desligar o fogo e retirar para batimento final.
Na mudança de cor: adicionar canela, ácido cítrico e sorbato dissolvidos em água.
Bater na batedeira velocidade média ~5min. Colocar em formas untadas em local seco até o dia seguinte.
⏱ ~5min + até o dia seguinte

🔄 Como funciona a sincronização
☁️
"Sincronizar agora" — envia todos os dados do dispositivo atual para o Google Sheets na nuvem. Use ao terminar de trabalhar ou após lançamentos importantes.
⬇️
"Carregar dados da nuvem" — substitui os dados locais pelos da nuvem. Use sempre ao abrir o app em um novo dispositivo ou após outra pessoa ter sincronizado.
⚠️
Regra de ouro ao usar em 2 dispositivos: sempre carregue da nuvem antes de começar a trabalhar, e sincronize ao terminar. Isso evita dados sendo sobrescritos.
📱
Primeiro acesso em novo dispositivo: configure a URL do Apps Script → Salvar → Carregar dados da nuvem → pronto!
🔧 Como atualizar o app (HTML)
⚠️ Este guia é APENAS para atualizar o arquivo HTML. Não mexe no Apps Script nem na sincronização.
1
Baixe o novo arquivo sabores_app.html gerado pelo Claude.
2
Acesse github.com, entre no repositório sabores-da-roca e clique no arquivo sabores_app.html.
3
Clique nos três pontinhos ··· no canto superior direito → Delete file → Commit changes.
4
Volte ao repositório → Add file → Upload files → arraste o novo arquivo → Commit changes.
5
Aguarde 1-2 minutos. No computador pressione Ctrl+F5. No celular limpe o cache do Chrome.
6
Verifique se o app abriu corretamente. Se sim, seus dados continuam intactos — eles ficam no Google Sheets, não no HTML.
⚙️ Como atualizar o Apps Script
⚠️ Este guia é APENAS para o Apps Script. Faça isso quando o Claude indicar que é necessário atualizar o backend.
1
Acesse script.google.com e abra o projeto Sabores da Roça.
2
Selecione todo o código atual (Ctrl+A) e delete. Cole o novo código fornecido pelo Claude.
3
Clique em 💾 Salvar (Ctrl+S).
4
Clique em Implantar → Gerenciar implantações → clique no lápis ✏️ para editar.
5
Em "Versão", selecione "Nova versão" — OBRIGATÓRIO para as mudanças entrarem em vigor.
6
Confirme que "Quem tem acesso" está como "Qualquer pessoa". Clique em Implantar.
7
A URL permanece a mesma. Volte ao app → Testar conexão → deve aparecer ✅.
8
Clique em Sincronizar agora para subir os dados com o novo código.
🔄 Como sincronizar entre dispositivos
ℹ️ Este guia é APENAS para manter os dados iguais em todos os dispositivos. Não altera o app nem o Apps Script.
1
Regra de ouro: sempre Carregar da nuvem antes de começar a trabalhar em qualquer dispositivo.
2
Ao terminar de trabalhar → Sincronizar agora para enviar os dados para a nuvem.
3
NUNCA trabalhe nos dois dispositivos ao mesmo tempo sem sincronizar — o último que sincronizar sobrescreve o outro.
4
Após sincronizar, verifique a mensagem de confirmação. Ela mostra quantos clientes, vendas e consignados foram enviados.
5
Se aparecer erro de conexão, verifique a internet e tente novamente. Seus dados estão salvos localmente e não serão perdidos.
💾 Backup e segurança dos dados
✅ Faça backup após cada sábado de rota e após atualizações importantes.
O que são seus dados e onde ficam:
Seus dados (clientes, vendas, repasses etc.) ficam em dois lugares: no localStorage do dispositivo (memória do navegador) e no Google Sheets (nuvem). O arquivo HTML é apenas a interface — não contém dados.
1
Backup JSON (dados completos): Role para cima nesta tela → clique em "💾 Exportar backup". Salve o arquivo .json gerado.
2
Onde salvar o backup: salve em pelo menos 2 locais — Google Drive + pasta no computador. Nunca dependa de um único local.
3
Backup do HTML: baixe também o arquivo sabores_app.html do GitHub e salve junto com o JSON. Assim você tem tudo para restaurar.
4
Frequência recomendada: após cada sábado de rota, após cadastrar muitos clientes e antes de qualquer atualização do sistema.
5
Para restaurar um backup: abra o app → Config → role até "Importar backup" → selecione o arquivo .json → confirme. Todos os dados serão restaurados.
⚠️ O Google Sheets guarda automaticamente o último estado sincronizado. Se perder o dispositivo, basta instalar o app em outro e carregar da nuvem.
